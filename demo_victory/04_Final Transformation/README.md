# LinkML Mapping

## Explanation of the Mapping
Root Alignment: The ZenodoDeposit class maps its title and description directly to the name and description fields found in your RO-Crate root dataset (@id: "./").

Creator Transformation: The Creator class maps the RO-Crate Person object's name to the Zenodo name field, and the identifier (specifically your ORCID) to the orcid field required by Zenodo for persistent identity.

Fixed Values: Fields like upload_type are set via equals_expression to "dataset", ensuring every record generated by this mapping complies with Zenodo's deposit requirements for your research.

This YAML file serves as the formal specification for your normalization layer. You can now use this definition within a Python environment (using linkml-runtime) to programmatically convert your ro-crate-metadata.json file into the precise format accepted by the Zenodo Sandbox API.
